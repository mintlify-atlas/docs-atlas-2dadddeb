## Generate Flashcards

Generate a flashcard deck from study content using AI. Supports text input and file uploads.

<Info>
Authentication is optional. Authenticated users can save decks and track study progress with spaced repetition.
</Info>

```bash POST /api/flashcards/generate
curl -X POST https://api.inspir.uk/api/flashcards/generate \
  -H "Content-Type: multipart/form-data" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -F "content=Photosynthesis is the process..." \
  -F "deckName=Biology Chapter 5" \
  -F "sourceName=Textbook" \
  -F "cardCount=15"
```

### Request Parameters

<ParamField body="content" type="string" required>
  Text content to generate flashcards from. Can be provided directly or via file upload.
</ParamField>

<ParamField body="file" type="file">
  Optional file upload (PDF, DOCX, TXT). Maximum 10MB. File content takes precedence over text content.
</ParamField>

<ParamField body="deckName" type="string">
  Name for the flashcard deck. If not provided, AI will suggest a name based on content.
</ParamField>

<ParamField body="sourceName" type="string">
  Source of the content (e.g., "Chapter 5", "Lecture Notes")
</ParamField>

<ParamField body="cardCount" type="number">
  Number of flashcards to generate. Min: 5, Max: 50, Default: 10
</ParamField>

### Response

<ResponseField name="success" type="boolean">
  Whether the operation was successful
</ResponseField>

<ResponseField name="deck" type="object">
  The generated flashcard deck
  
  <Expandable title="Deck Object">
    <ResponseField name="id" type="string">
      Unique deck identifier (null for unauthenticated users)
    </ResponseField>
    
    <ResponseField name="deckName" type="string">
      Name of the deck
    </ResponseField>
    
    <ResponseField name="sourceName" type="string">
      Source name
    </ResponseField>
    
    <ResponseField name="cards" type="array">
      Array of flashcards
      
      <Expandable title="Card Object">
        <ResponseField name="id" type="string">
          Unique card identifier (UUID)
        </ResponseField>
        
        <ResponseField name="front" type="string">
          Question or prompt text
        </ResponseField>
        
        <ResponseField name="back" type="string">
          Answer text (2-4 sentences)
        </ResponseField>
        
        <ResponseField name="explanation" type="string">
          Optional context, mnemonic, or additional info
        </ResponseField>
      </Expandable>
    </ResponseField>
    
    <ResponseField name="cardCount" type="number">
      Total number of cards in the deck
    </ResponseField>
  </Expandable>
</ResponseField>

### Error Responses

<Expandable title="400 Bad Request">
```json
{
  "error": "Content is required"
}
```

```json
{
  "error": "Failed to process file",
  "details": "File format not supported"
}
```

```json
{
  "error": "Invalid file type. Only PDF, DOCX, and TXT files are allowed."
}
```
</Expandable>

<Expandable title="500 Internal Server Error">
```json
{
  "error": "Failed to generate flashcards",
  "details": "The AI returned an invalid format"
}
```
</Expandable>

## Record Study Progress

Record the results of a study session using the SM-2 spaced repetition algorithm.

<Warning>
This endpoint requires authentication.
</Warning>

```bash POST /api/flashcards/deck/:id/progress
curl -X POST https://api.inspir.uk/api/flashcards/deck/123e4567-e89b-12d3-a456-426614174000/progress \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "results": [
      {
        "cardId": "card-uuid-1",
        "quality": 4,
        "correct": true
      },
      {
        "cardId": "card-uuid-2",
        "quality": 2,
        "correct": false
      }
    ],
    "mode": "flip",
    "durationSeconds": 300
  }'
```

### Path Parameters

<ParamField path="id" type="string" required>
  The unique identifier of the deck
</ParamField>

### Request Parameters

<ParamField body="results" type="array" required>
  Array of card results from the study session
  
  <Expandable title="Result Object">
    <ParamField body="cardId" type="string" required>
      Card ID
    </ParamField>
    
    <ParamField body="quality" type="number" required>
      Quality of recall (0-5):
      - 0: Complete blackout
      - 1: Incorrect, but familiar
      - 2: Incorrect, but close
      - 3: Correct with difficulty
      - 4: Correct with hesitation
      - 5: Perfect recall
    </ParamField>
    
    <ParamField body="correct" type="boolean" required>
      Whether the answer was correct
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="mode" type="string">
  Study mode used
</ParamField>

<ParamField body="durationSeconds" type="number">
  Duration of the study session in seconds
</ParamField>

### Response

<ResponseField name="success" type="boolean">
  Whether the operation was successful
</ResponseField>

<ResponseField name="updates" type="array">
  Array of updated progress for each card with new review intervals
</ResponseField>

<ResponseField name="stats" type="object">
  Updated mastery statistics
</ResponseField>

<ResponseField name="summary" type="object">
  Session summary
  
  <Expandable title="Summary Object">
    <ResponseField name="cardsStudied" type="number">
      Total cards studied
    </ResponseField>
    
    <ResponseField name="cardsCorrect" type="number">
      Number of correct answers
    </ResponseField>
    
    <ResponseField name="accuracy" type="number">
      Accuracy percentage (0-100)
    </ResponseField>
  </Expandable>
</ResponseField>

## Spaced Repetition Algorithm

<Info>
The flashcards feature uses the SM-2 (SuperMemo 2) spaced repetition algorithm to optimize learning:

- **Quality Ratings (0-5)**: After each review, rate how well you recalled the answer
- **Ease Factor**: Adjusts based on performance (easier cards have longer intervals)
- **Intervals**: Time between reviews increases exponentially for well-known cards
- **Mastery Levels**: Cards progress through learning stages as you review them correctly

This ensures you review cards at optimal intervals for long-term retention.
</Info>
